---
description: 최근 코드 변경사항을 관련 문서에 반영합니다
---

방금 변경한 코드를 기반으로 관련 문서를 업데이트해줘.

## 문서 업데이트 절차

### 1. 변경 범위 파악
먼저 최근 수정된 파일들을 확인하고 어떤 문서가 영향을 받는지 판단해줘.

```bash
git diff --name-only HEAD
```

### 2. 업데이트 대상별 규칙

#### API Route 추가/변경 → `docs/api/` 업데이트
- `docs/api/` 디렉토리가 없으면 생성
- 파일명: `docs/api/[도메인].md` (예: `docs/api/donations.md`)
- 형식:
```markdown
## [HTTP메서드] /api/[경로]

**인증**: 필요 / 불필요 (Bearer Token)
**권한**: 일반 사용자 / 관리자

### 요청
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| field | string | Y | 설명 |

### 응답 (200)
```json
{ "success": true, "data": {} }
```

### 에러
| 코드 | 상황 |
|------|------|
| 400 | 유효성 검사 실패 |
| 401 | 미인증 |
```

#### DB 스키마 변경 → `DB_CONNECTION_GUIDE.md` 업데이트
- 새 테이블이 추가된 경우 테이블 목록 섹션에 추가
- 관계 변경 시 ERD 텍스트 설명 업데이트

#### 이슈 해결 → 이슈 리스트 체크
- `ISSUE_LIST.md` 또는 `ADMIN_ISSUE_LIST.md`에서 해결된 항목에 체크
- `- [ ]` → `- [x]`

#### 환경변수 추가 → `README.md` 업데이트
- `.env.example` 파일이 있으면 해당 파일도 업데이트

#### 새 컴포넌트/페이지 → JSDoc 주석
- 컴포넌트 파일 상단에 간단한 JSDoc 추가 (별도 문서 파일 대신)
```typescript
/**
 * XxxComponent - [컴포넌트 설명]
 * @param props.fieldName - [설명]
 */
```

### 3. 실행
위 규칙에 따라 필요한 문서를 직접 업데이트하거나 생성해줘.
변경한 문서 목록을 마지막에 요약해서 알려줘.
